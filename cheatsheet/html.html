What is html? consists of?

  • html (hyperText Markup Language).

What are Elements and their types?

Each Element: Start tag, content, end tag.

Two types: Block level and inline level.

Define/list some of the Block level Elements?

They form a new block on the page â€“ there will be a blank line before and after each block level elements.

  • <p>
  • <h1>, <h2>, <h3>, <h4>, <h5>, <h6>
  • <ol>, <ul>
  • <pre>
  • <div>
  • <br>

What worth knowing about heading level Elements <h1>, <h2>, <h3>, <h4>, <h5>, <h6>?

  • Their font size decreases from H1 up to H6.
  • Their semantic importance decreases from H1 up to H6.
  • They should not be used for styling font in bold but to emphasize the importance of heading.
  • Avoid using H1 more than once in a page.
  • Avoid using h2 to h3 or h4 to list items and let ordered or unordered list take care of that and reserve them to headings and subheadings with combination use of the p tag.

What worth knowing about <div>?

  • div element is purely structural.
  • Use it to group blocks of related elements.
  • Grouping making styling easier later.
  • It is preferable to use with it the unique id attribute instead of the class attribute.
  • when used with id attribute you can divide your page into header content and footer.

div Example:


            <div id="header">
            <p>Article heading</p>
            <p>Home : Articles : Article one</p>
            </div>
            

            <div id="content">
            <p>Some content.</p>
            <p>Some more content.</p>
            </div>
            

            <div id="footer">
            <p>Author: Nik</p>
            </div>

What worth knowing about <span>?

  • span is also purely structural.
  • Like div, but for inline content.
  • Grouping making styling easier later.
  • It is preferable to use with the class attribute rather than with the unique ID attribute.
  • When used with the class attribute you can manipulate with precision many parts of your single div or p through JS and/or CSS.

Span Example:


            <div id="header">
            <p>Article heading</p>
            <p><span class="nav">Home</span> :
            <span class="nav">Articles</span> :
            Article one</p>
            </div>
            

            <div id="content">
            <p>Some content.</p>
            <p>Some more content.</p>
            </p>
            

            <div id="footer">
            <p>Author:
            <span class="author-name">Nik</span></p>
            </div>

What does Elements Nesting mean?

Elements can be placed inside other elements. We say that they wrapped (or nested) inside each other.

Elements Nesting Example:


            <p>This is an <em>html</em> paragraph.</p>

Result:

This is an html paragraph.

Caution! Elements must not overlap. You can not close an element unless all the elements that it contains are closed first.
Inline Elements can nest inside block Elements: e.g em inside p.
Inline ELements can nest inside inline Elements e.g. em inside strong.
Block Elements sometimes cannot nest inside another Block Element e.g h inside p.
Block Elements sometimes can nest inside another block ELement e.g div inside div or ul inside div or p inside div.
Each element has different rules about what can and cannot be placed inside it. so aformentioned nesting cases are broadly correct. We can use MDN to see what those rules are for any element we're interested in. we can search for [mdn element ul], then search for "Permitted content" and there we can tell what elements can be nested inside the ul element which may vary among elements.

What are the types of images?

  • PNG (Portable Network Graphics), .png
    • Raster format.
    • Lossless.
    • Best for text, line-art, flat colors.
  • JPEG (Joint Photographic Expert Group), .jpeg, .jpg
    • Raster format.
    • Lossy.
    • Good for photos.
  • SVG (Scalable Vector Graphics), .svg.
    • Vector format.
    • Supports interactivity and animation.

Do different elements have different attributes?

YES! The html specification says which attributes are allowed for each element, you can't just write an attribute on an element and hope it will work.

              <!-- This does *not* work -->
              <p src="name-of-file.png">...</p>

What is an empty element?

HTML elements with no content are called empty elements.
<br> is an empty element without a closing tag (the <br> tag defines a line break).
Empty elements can be "closed" in the opening tag like this: <br />.
HTML5 does not require empty elements to be closed. But if you want stricter validation, or if you need to make your document readable by XML parsers, you must close all HTML elements properly..
